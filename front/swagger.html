Autorizações

A api tem um sistema de autorização na maioria das rotas, para acessa-las é necessário fornecer o AccessToken.

Como conseguir um AcessToken?

O accessToken é obtido no login. Faça login na aplicação pela rota "POST /usuarios/login", aperte o botão "Try it out" e depois aperte o botão azul "Execute"

O login será feito com a conta de teste padrão:

{

"email": "teste@teste.com",

"senha": "123456"

}

Receberemos em resposta os dados abaixo:

Response body:

{

"refreshToken": "gekasvsafdngasldnkasdfWI",

"nomeUsuario": "Usuário Teste"

}

Response headers:

access-control-allow-headers: Origin,X-Requested-With,Content-Type,Accept

access-control-allow-origin: *

authorization: eghryukfghdfgthryutuijrtnjetryjrtyjtrythreth

connection: keep-alive

content-length: 100

content-type: application/json; charset=utf-8

date: Thu,16 Feb 2023 11:31:14 GMT

etag: W/"64-JmbynJoKQROt50gYDTpomXrmWBk"

keep-alive: timeout=5

x-powered-by: Express

Note que recebemos o AccessToken no Headers > authorization

E tambem recebemos o RefreshToken no Body > "refreshToken"

Autorizando todas as Rotas com o AccessToken

Copie códido do AcessTokem. Aperte o botão verde "Authorize" que esta no canto direito acima das rotas.

Cole o codigo e aperte o botão "Authorize" e depois o botão "Close"

Agora todas as rotas estão autorizadas!

Usando o RefreshToken

Quando o AccessToken expira precisamos renova-lo. Copie o refreshToken recebido no login.

Use a rota "POST /usuarios/atualiza_refresh", aperte o botão "Try it out", cole o refreshToken no Body e depois aperte o botão azul "Execute"

Em resposta receberemos um novo AccessToken e também um novo refreshToken. Faça novamente a autorização com o AccessTokem
